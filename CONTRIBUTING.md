## Development Workflow
1. Create a branch from `main`
2. Install dependencies: `python -m pip install -e .[dev]`
3. Run checks locally: `ruff check .` and `pytest -q`
4. Open a PR with:
   - Problem statement
   - Summary of changes
   - Validation evidence

## Support
For issues or collaboration requests, open a GitHub issue and tag `@Muhammad-Farooq-13`.

## Commit Message Style
Use conventional commits:
- `feat:` new functionality
- `fix:` bug fix
- `docs:` documentation only
- `refactor:` internal restructuring
- `test:` test updates
